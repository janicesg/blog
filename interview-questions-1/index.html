Introduction

This post records my detailed notes when I did my routine review on Java related concepts and questions, hopefully this is helpful to you.

What is your impression of java?

  1. Why do we need java? Java “write once, run anywhere”, GC
  2. What are the ecosystem of java? big data, app dev, compiler optimization, java optimization
  3. What are the good things about java?
  • OOP
  • Generics
  • reflection: 反射就是在运行时才知道要操作的类是什么，并且可以在运行时获取类的完整构造，并调用对应的方法。 (ref: https://www.cnblogs.com/chanshuyi/p/head_first_of_reflection.html) - security concern, used for frameworks.
  1. What about jvm: jvm, jit, aot,tools

What is the difference between Error and Exception?

  1. Exception and Error both inherits Throwable, which is the basic type of java exception handling.
    Exception is expected, and it can and should be catched and processed. Error is the unlikely situation in normal execution, most of the errors can cause JVM to fall into an abnormal state. It is hard and not necessary to catch. e.g. OutOfMemoryError.
    There are unchecked and checked exceptions. Unchecked exceptions happens while executing, like NPE, ArrayIndexOutofBoundsException. checked exceptions have to be wrote explicitly in the code, which compiler will check.
  • Errors:LinkageError, NoClassDefFoundError, UnsatisfiedLinkError, ExceptionInitializeError, VirtualMachineError, OutOfMemoryError, StackOverflowError
  • Exceptions: IOException, RuntimeException, NPE, ClassNotFoundException
  1. what is the difference between ClassNotFoundException and NoClassDefFoundError?
    ClassNotFoundException happens when java using reflection class.forName to do class loading, and a class name is passed as a parameter to this method, then this class will be added to the JVM memory. if it cannot be found in the classpath, then it will throw ClassNotFoundException.
    NoClassDefFoundError will be thrown when JVM or classloader tries to load a class but cannot find the definition of the class, though it was present when compiling but not when executing. it could be cuased by missing the class while compiling or the jar is damaged.
  2. 4 Principles when handling Exceptions and Errors
  • Don’t catch big exceptions, must catch as specific as possible
  • Don’t swallow the exception and don’t handle it properly in catch phrase
  • Throw early
  • catch late: keep the original cause, throw a new exception or the same exception

What is java class loader?

(ref: https://www.cnblogs.com/chanshuyi/p/jvm_serial_07_jvm_class_loader_mechanism.html)

  1. Identify the initial value of class variables: at preparation stage of classloading, JVM will initialize class variables with default values. If the variable is static final, then it will be set as the desired value.
  2. Entry of initialization: at initialization stage of classloading, JVM will find the main() as the entry point, and initialize the class where main() belongs to. When JVM needs to initialize a class, it will start with class initializer, then instance intializer.
  3. Class initializer: JVM will collect the class variables, static code in sequence.
  4. Instance initializer: JVM will collect the instance variables, normal code block, and constructor code in sequence.
  5. Conditions which trigger initialization:
  • when new(), getstatic(),putstatic(),invokeStatic() is called, if the class is not initialized, then trigger the initialization. It will happpen when create a new instance using new(), set or read a static variable, invoke a static method.
  • if the parent is not initialized, u need to initialize the parent first
  • when the jvm starts, it first initializes the class where the main() exists

what is final, finally, and finalize?

  1. final is an modifier, final variable means the value cannot be modified, final class means it cannot be inherit or implement, final method menas no overriding.
  2. finally is a clause to force execute, e.g. we can use try-catch-finally or try-finally to close jdbc connection or release the lock.
  3. finalize is a method of Object, used for garbage collection, not recommend.

What happens from bytecode to machine code?

  1. Java compiler: javac compiles java code to bytecode
  2. Java Interpreter: translates bytecode to machine code. start up fast but run slow / Just-in-Time Interpreter: runtime compilation (c1 mode and c2 mode), start slow, but run fast afterwords
  3. Ahead-Of-Time compiler: direct from java code to bytecode, but more errors
  4. in terms of compile speed: interpreter > AOT > JIT; compile quality : JIT > AOT > interpreter

What is Garbage Collection?

  1. Mechanism: GC Root Tracing, JVM memory is split into regions: Young(eden,from,to 8:1:1), Old Generation
  2. Reachability analysis:
  3. algorithm: Mark Sweep, copying, mark-compact algorithm.
  4. Garbage Collector:
  • Serial Collector: single thread, stop the world.-xx:+UseSerialGC
  • Parallel GC: multiple threads, stop the world.
  • Concurrent Mark Sweep GC: allocate more CPU to ensure application throughput, multithreading to scan the heap memory to mark instances for eviction and then sweep the marked instances, xx:+UseParNewGC
  • G1 GC: it separates the heap memory into regions, and does collection within them in parallel. G1 also does compacts the free heap space on the go just after reclaiming the memory.

What is Strong , Soft , weak , phantom reachable?

  • Strong: when an object can be reached without any other ref, e.g. new instance.
  • Soft: exampt some objects from GC, only when no memory in JVM, it will be GC.
  • weakly: when an object can only be reached via weak ref, nearly in finalize
  • phantom: reference queue

What is String, StringBuffer and StringBuilder?

  • String is Immutable, finalize, thread-safe, use a pool to reuse the common chars. Try to avoid ‘+’. only String a = “abc” is added to the pool, if String a = new String(“abc”) is not added.
  • StringBuffer is thread-safe for modifying string values. it is better to estimate the size of the string when initialize the buffer or builder, otherwise arraycopy is very costly.
  • StringBuilder is not thread-safe.

int VS Integer?

始数据类型和 Java 泛型并不能配合使用，也就是Primitive Types 和Generic 不能混用，于 是JAVA就设计了这个auto-boxing/unboxing机制，实际上就是primitive value 与 object之间的隐式转换机制，否则要是没有这个机制，开发者就必须每次手动显示转换，那多麻烦是不是? 但是primitive value 与 object各自有各自的优势，primitive value在内存中存的是值，所以找到primitive value的内存位置，就可以获得值;不像object存的是reference，找到object的 内存位置，还要根据reference找下一个内存空间，要产生更多的IO，所以计算性能比primitive value差，但是object具备generic的能力，更抽象，解决业务问题编程效率高。于是JAVA设计 者的初衷估计是这样的:如果开发者要做计算，就应该使用primitive value如果开发者要处理业务问题，就应该使用object，采用Generic机制;反正JAVA有auto-boxing/unboxing机制， 对开发者来讲也不需要注意什么。然后为了弥补object计算能力的不足，还设计了static valueOf()方法提供缓存机制，算是一个弥补

ArrayList VS Vector VS LinkedList?

  1. ArrayList and Vector are achieved by array, LinkedList via doubly linked list
  2. if exceeds capacity, arrayLIst will do arrayCopy, and increase 50%, but vector will increase to 2 times.
  3. thread-safe: vector
  4. arraylist read fast, delete and add slow. but linkedlist read slow
