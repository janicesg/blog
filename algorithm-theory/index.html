Introduction

This post includes my notes on watching leetcode tutorials, hopefully it is helpful to you.

Practice strategy

10000 hour principle:

  1. Chunk it up : Split the topics, and do it one by one systematically
  2. Deliberate practice: it is common to find it uncomfortable, boring, especially in unfamiliar area.
  3. Actively seek feedback to improve the code quality, which includes
  • Instant feedback: from interviewers
  • Active feedback: Inspect solutions from leetcode and discussion panel
  • Passive feedback: Ask for code review from peers

Things to remember when practice

  1. Ask for Clarifications, remember that it is ok to ask questions in code interviews to find out edge cases and special requirements
  2. Think about all possible solutions, find the best one with lowest complexity. Present all your work clearly to the interviewer, including comparison on time & space, and how to optimize it.
  3. Code, Code, Code !
  4. Write Test cases

Time Complexity for Binary Search, Binary Tree Tranversal, Optimal Sorted Matrix search, Merge sort

Algorithm Runtime Complexity
Binary Search O(log n)
Binary tree traversal O(n)
Optimal sorted matrix search O(log n)
merge sort O(nlog n)

Data Structures: Array, Stack, Queue, LinkedList, Set, Tree, Graph

Array (Time complexity)

  • access : O(1)
  • insert (average): O(n) , you need to move backward all the elements behind the new element
  • delete (average): O(n) , you need to move forward all the elements behind the new element

Linked List

Advantage:

  • Applicable for operations with many inserts and deletions
  • When not sure about how many elements, e.g. Blockchain.

Complexity:

  • access: O(n)
  • insert (average): O(1) , you need to change 2 links
  • delete (average): O(1) , you need to change 2 links

Tricks: Use pointers: e.g. faster and slow pointer to detect cycles in linked list

Stack and Queue

Useful link: https://www.bigocheatsheet.com/

  • STACK - FILO : Array or Linked List
  • Queue - FIFO : Array or Double Linked List
  • Test problem : Valid parentheses https://leetcode.com/problems/valid-parentheses/
  • number 232, 225 ( use queue to write stack)

Tricks: Use more stacks / queues

Priority Queue

Scenario: when elements are weighted differently
Def: insert sequentially, but pop based on priority.
Mechanism: Heap( Binary, Binomial, Fibonacci) or BST

  • Heap: Mini , Max
  • Test problem: 703 (max K in N numbers - min heap), 239(sliding window - max heap / deque)

HashSet and HashMap

  1. HashFunction: key -> value; handle collision: save the elements in linked list

  2. abstract data structure: List VS Map Vs Set

    Structure repeat elements insert search
    List(Array or list) true O(1) O(n)
    Map false O(1) O(1)
    Set (HashSet or Tree: sorted) false O(1) O(1) or O(log(n))

  3. map and set are commonly used for searching and counting

Tree, Binary Tree, Graph

  1. graph – no loop –> binary tree – one child-> linked list
  2. binary search tree = ordered search tree.
    leetcode 236 : findPorQ(root,p,q) recursion
  3. pre-order(root-left-right), in-order(l-root-right), post-order(l-r-root) traversal
